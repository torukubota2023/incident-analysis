# インシデント・アクシデント報告システム 院内LAN展開 依頼書

**発信元**: 医療安全委員会 / 副院長
**宛先**: 情報システム部門 御中
**日付**: 2026年3月
**件名**: インシデント・アクシデント報告システムの院内ネットワーク導入について

---

## 1. 依頼の背景

現在、当院のインシデント・アクシデント報告は**紙の報告書**で運用しています。

### 現行の紙運用における課題

| # | 課題 | 影響 |
|---|---|---|
| 1 | **報告に時間がかかる** | 1件あたり10〜15分。記入の手間から報告が後回しになり、報告漏れが発生 |
| 2 | **集計が手作業** | 月次集計に毎回数時間。Excelへの転記ミスも発生 |
| 3 | **分析が困難** | 部署別・カテゴリ別のクロス分析、再発パターン検出が手作業では実質不可能 |
| 4 | **リアルタイム把握ができない** | 月末まで件数推移が分からず、対策が後手に回る |
| 5 | **紙の保管・検索の負担** | 過去の報告書を探すのに時間がかかる。保管場所も必要 |
| 6 | **委員会資料作成の負担** | 毎月の医療安全委員会向け資料を手作業で作成 |

---

## 2. 導入の目的

上記課題を解決するため、**Webベースのインシデント報告・分析システム**を院内LANに導入します。

### 期待される効果

| 目的 | 具体的な効果 |
|---|---|
| **報告の効率化** | 入力時間を10〜15分 → **3分以内**に短縮。スマホからも入力可能 |
| **報告率の向上** | 簡便な入力UIにより、報告のハードルを下げレベル0〜1の報告を増加 |
| **自動集計** | 月別・部署別・カテゴリ別・重症度別の集計が**リアルタイムで自動化** |
| **分析の高度化** | クロス集計・再発パターン検出・前月比/前年同月比の比較が即時可能 |
| **指標の可視化** | 1000患者日あたりインシデント率・転倒転落率を自動算出 |
| **委員会資料の効率化** | ダッシュボードをそのまま委員会で投影、CSVダウンロードでExcel加工も可能 |

### 導入方針
- **院内LANのみの閉域運用**（インターネット不要）
- **患者氏名・IDは入力しない設計**（個人情報保護）
- データはサーバーPC内のCSVファイルに保存（外部送信なし）
- 既存の院内ネットワーク・PC設備をそのまま活用（追加ハードウェア購入不要）
- **無料のオープンソースソフトウェアのみ使用**（ライセンス費用ゼロ）

## 3. システム概要

### 構成

```
院内LAN
┌──────────────────────────────────────────┐
│                                          │
│  【サーバーPC】1台                         │
│   既存のWindows PC（常時起動）             │
│   Python + Streamlit（無料）              │
│   ポート: 8501                            │
│                                          │
│     ↕ HTTP通信（院内LANのみ）             │
│                                          │
│  【クライアント】複数台                     │
│   ナースステーション・医局・安全管理室      │
│   ブラウザのみ（ソフトインストール不要）     │
│   PC・タブレット・スマホ対応               │
│                                          │
└──────────────────────────────────────────┘
```

### アプリの機能一覧

| 機能 | 内容 |
|---|---|
| 報告入力 | Webフォームから入力（スマホ対応、3分以内目標） |
| ダッシュボード | KPI表示・月次推移・カテゴリ別・部署別グラフ |
| 詳細分析 | クロス集計・時間帯分析・転倒転落専門・再発分析 |
| データ一覧 | 全報告の閲覧・フィルタ・CSVダウンロード |
| 使い方ガイド | アプリ内ヘルプ（重症度基準表含む） |
| デモモード | サンプルデータで機能を体験（実データと混在しない） |

## 4. 情報システム部門への依頼事項

### 4-1. サーバーPCの準備

| 項目 | 要件 | 備考 |
|---|---|---|
| PC | Windows 10/11、メモリ4GB以上 | 既存の常時起動PCで可 |
| 固定IP | DHCPリザベーション or 固定IP設定 | 各端末のブックマーク用に必要 |
| 電源管理 | スリープ無効設定 | 常時アクセス可能にするため |
| UPS | 可能であれば接続 | 停電時のデータ保護 |

### 4-2. ネットワーク設定

| 項目 | 内容 |
|---|---|
| ファイアウォール | サーバーPCのポート8501を院内LANに開放（プライベートネットワークのみ） |
| Wi-Fi | 院内Wi-Fiからも同一サブネットでアクセスできることを確認（スマホ入力用） |

### 4-3. ソフトウェアインストール（サーバーPCのみ）

以下を順番にインストール・実行してください。**詳細手順は別添の「院内LAN展開マニュアル」を参照**。

1. **Python 3.10以上**をインストール（PATHにチェック必須）
2. **Git**をインストール（オプション。ZIPダウンロードでも可）
3. リポジトリをクローンまたはZIPを展開
4. 仮想環境の作成とパッケージインストール
5. `start_app.bat` のダブルクリックで起動確認
6. Windowsスタートアップに `start_app.bat` のショートカットを登録
7. タスクスケジューラに `backup.bat` を毎日23:00で登録

### 4-4. クライアント端末の設定（各端末）

- **ソフトウェアのインストールは不要**です
- ブラウザのブックマークに `http://[サーバーIP]:8501` を登録するだけ
- 必要に応じてQRコードを各ステーションに掲示

## 8. 補足: セキュリティに関する確認事項

| 項目 | 対応状況 |
|---|---|
| 患者個人情報 | 氏名・ID・生年月日は入力項目に含まれない。年齢・性別・病名のみ |
| 通信経路 | 院内LAN内のHTTP通信のみ。インターネットへの通信なし |
| データ保存場所 | サーバーPC内のCSVファイル。クラウド・外部サーバーへの送信なし |
| アクセス範囲 | 同一ネットワーク（院内LAN）上の端末のみアクセス可能 |
| ソフトウェア | Python・Streamlit（オープンソース、MITライセンス）。マルウェアリスクなし |
| バックアップ | 日次自動バックアップ。30日超の古いバックアップは自動削除 |
