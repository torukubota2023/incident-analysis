## 1. 概要

### このマニュアルでできること
- 院内の**1台のPC（サーバー）** でアプリを起動
- 同じネットワーク上の **複数のPC・タブレット・スマホ** からブラウザでアクセス
- インターネット接続は**不要**（院内LANのみで完結）
- データはサーバーPCの**CSVファイル**に保存（外部に出ない）

### 構成図

```
院内LAN（同一ネットワーク）
┌─────────────────────────────────────────────┐
│                                             │
│  【サーバーPC】                               │
│   Windows PC（常時起動）                      │
│   IP: 192.168.1.100（例）                    │
│   Streamlit アプリ稼働中（ポート8501）         │
│   データ保存: C:\incident-analysis\data\     │
│        │                                    │
│        ├──→ 【ナースステーションPC】           │
│        │     ブラウザで http://192.168.1.100:8501 │
│        │                                    │
│        ├──→ 【医局PC】                       │
│        │     ブラウザで http://192.168.1.100:8501 │
│        │                                    │
│        ├──→ 【スマホ・タブレット】             │
│        │     ブラウザで http://192.168.1.100:8501 │
│        │                                    │
│        └──→ 【医療安全管理室PC】              │
│              ブラウザで http://192.168.1.100:8501 │
│                                             │
└─────────────────────────────────────────────┘
```

## 2. 必要な環境

### サーバーPC（1台）
| 項目 | 要件 |
|---|---|
| OS | Windows 10 / 11（64bit） |
| メモリ | 4GB以上（推奨8GB） |
| ストレージ | 空き容量 2GB以上 |
| ネットワーク | 院内LANに有線接続（推奨） |
| 電源 | **常時起動**（スリープしない設定にする） |

### クライアント端末（アクセスする側）
- ブラウザがあればOK（Chrome / Edge / Safari）
- PC、タブレット、スマートフォンいずれも可
- 同じ院内ネットワークに接続されていること

### インストールするソフトウェア（サーバーPCのみ）
- **Python 3.10以上**（無料）
- **Git**（無料、オプション）

## 3. サーバーPCのセットアップ

### 3-1. Pythonのインストール

1. https://www.python.org/downloads/ にアクセス
2. 「Download Python 3.12.x」をクリック
3. インストーラーを実行
4. **⚠️ 重要: 「Add python.exe to PATH」に必ずチェック**を入れる
5. 「Install Now」をクリック

#### 確認方法
コマンドプロンプト（Windowsキー → `cmd` と入力 → Enter）を開いて：
```
python --version
```
`Python 3.12.x` と表示されればOK。

### 3-2. Gitのインストール（オプション）

GitHubからコードを取得する場合に必要です。

1. https://git-scm.com/download/win にアクセス
2. 「64-bit Git for Windows Setup」をダウンロード
3. インストーラーを実行（すべてデフォルト設定でOK）

## 4. アプリの配置と起動

### 方法A: GitHubからクローン（Gitインストール済みの場合）

コマンドプロンプトで以下を実行：

```bat
cd C:\
git clone https://github.com/torukubota2023/incident-analysis.git
cd incident-analysis
```

### 方法B: ZIPダウンロード（Gitなしの場合）

1. ブラウザで https://github.com/torukubota2023/incident-analysis にアクセス
2. 緑色の「Code」ボタン → 「Download ZIP」
3. ダウンロードしたZIPを `C:\` に展開
4. フォルダ名を `incident-analysis` に変更

### 4-1. 仮想環境の作成とパッケージインストール

コマンドプロンプトで以下を**1行ずつ**実行：

```bat
cd C:\incident-analysis

python -m venv venv

venv\Scripts\activate

pip install -r requirements.txt
```

各コマンドの意味：
| コマンド | 何をしている？ |
|---|---|
| `cd C:\incident-analysis` | アプリのフォルダに移動 |
| `python -m venv venv` | Python仮想環境を作成（他のソフトに影響しない） |
| `venv\Scripts\activate` | 仮想環境を有効化（プロンプトに`(venv)`が表示される） |
| `pip install -r requirements.txt` | 必要なライブラリをインストール |

#### よくあるエラー
| エラー | 対処 |
|---|---|
| `'python' は認識されていません` | Pythonインストール時にPATHにチェックを入れ忘れ。再インストール |
| `pip でエラーが出る` | `python -m pip install --upgrade pip` を先に実行 |

### 4-2. アプリの起動

```bat
cd C:\incident-analysis
venv\Scripts\activate
streamlit run app.py --server.port 8501 --server.address 0.0.0.0 --server.headless true
```

⚠️ **`--server.address 0.0.0.0`** が重要です。これにより他のPCからアクセス可能になります。

成功すると以下が表示されます：
```
You can now view your Streamlit app in your browser.

  Network URL: http://192.168.x.x:8501
  External URL: http://xxx.xxx.xxx.xxx:8501
```

**この `Network URL` が院内からアクセスするアドレスです。**

### 4-3. Windowsファイアウォールの許可

初回起動時にWindowsファイアウォールの警告が出ます：

1. 「プライベートネットワーク」に**チェック**を入れる
2. 「パブリックネットワーク」は**チェックを外す**
3. 「アクセスを許可する」をクリック

もし警告が出なかった場合、手動で設定：
1. Windowsキー → 「ファイアウォール」で検索 → 「Windows Defender ファイアウォール」
2. 「アプリにWindows Defenderファイアウォール経由の通信を許可する」
3. 「設定の変更」→「別のアプリの許可」
4. `C:\incident-analysis\venv\Scripts\python.exe` を追加
5. 「プライベート」にチェック → OK

## 5. 他のPCからアクセスする方法

### 5-1. サーバーPCのIPアドレスを確認

サーバーPCのコマンドプロンプトで：
```bat
ipconfig
```

「IPv4 アドレス」の値を確認（例: `192.168.1.100`）

### 5-2. クライアント端末からアクセス

ブラウザのアドレスバーに入力：
```
http://192.168.1.100:8501
```
（IPアドレスはサーバーPCのものに置き換え）

### 5-3. ブックマーク登録を推奨

各端末のブラウザでブックマーク（お気に入り）に登録しておくと便利です。

### 5-4. スマホ・タブレットからのアクセス

1. 院内Wi-Fiに接続（院内LANと同一ネットワークであること）
2. ブラウザで `http://192.168.1.100:8501` にアクセス
3. ホーム画面に追加すればアプリのように使える

#### QRコード活用
サーバーURLのQRコードを各ステーションに掲示すると便利です。

## 6. 自動起動の設定

PCの再起動後もアプリが自動で起動するように設定します。

### 6-1. 起動バッチファイルの作成

`C:\incident-analysis\start_app.bat` は同梱済みです。内容：

```bat
@echo off
echo ========================================
echo  インシデント報告システム 起動中...
echo ========================================
cd /d C:\incident-analysis
call venv\Scripts\activate
streamlit run app.py --server.port 8501 --server.address 0.0.0.0 --server.headless true
pause
```

ダブルクリックでアプリが起動します。

### 6-2. Windows起動時に自動実行

1. `Windowsキー + R` → `shell:startup` と入力 → Enter
2. 開いたフォルダに `start_app.bat` の**ショートカット**を作成
   - `start_app.bat` を右クリック →「ショートカットの作成」
   - 作成されたショートカットをスタートアップフォルダにコピー

これでPC起動時に自動でアプリが立ち上がります。

### 6-3. スリープしない設定

サーバーPCは常時起動が必要です：

1. 設定 → システム → 電源とスリープ
2. 「スリープ」→ **「なし」** に設定
3. 「画面」→ 必要に応じてオフにしてもOK

## 7. データのバックアップ

### データの保存場所
```
C:\incident-analysis\data\incidents.csv
```

### 7-1. 手動バックアップ
`incidents.csv` を定期的にUSBメモリやネットワークドライブにコピー。

### 7-2. 自動バックアップ（バッチファイル）

`C:\incident-analysis\backup.bat` は同梱済みです。内容：

```bat
@echo off
set BACKUP_DIR=C:\incident-backup
set DATE=%date:~0,4%%date:~5,2%%date:~8,2%

if not exist %BACKUP_DIR% mkdir %BACKUP_DIR%

copy "C:\incident-analysis\data\incidents.csv" "%BACKUP_DIR%\incidents_%DATE%.csv"

echo バックアップ完了: %BACKUP_DIR%\incidents_%DATE%.csv
```

### 7-3. タスクスケジューラで自動実行

1. Windowsキー → 「タスクスケジューラ」で検索
2. 「基本タスクの作成」
3. 名前: 「インシデントデータバックアップ」
4. トリガー: 「毎日」→ 時刻を設定（例: 23:00）
5. 操作: 「プログラムの開始」→ `C:\incident-analysis\backup.bat`

## 8. セキュリティ注意事項

### ⚠️ 重要な注意事項

| 項目 | 対策 |
|---|---|
| **患者個人情報** | アプリには患者氏名・IDを**入力しない**設計です。年齢・性別・病名のみ |
| **ネットワーク** | 院内LANのみで運用。外部（インターネット）からはアクセス不可 |
| **ファイアウォール** | 「プライベートネットワーク」のみ許可。パブリックは不可 |
| **データ保存** | CSVファイルはサーバーPCのローカルに保存。クラウドには送信されません |
| **アクセス制限** | 同一ネットワーク上の全端末からアクセス可能。必要に応じてIPフィルタを検討 |
| **暗号化** | HTTP通信（暗号化なし）。院内LANクローズド環境での運用を前提 |

### Streamlit Cloud版との使い分け

| 用途 | 推奨環境 |
|---|---|
| 機能デモ・説明用 | Streamlit Cloud（デモモード） |
| 実データ入力・分析 | **院内LAN版**（このマニュアル） |
| 委員会報告資料作成 | 院内LANでCSVダウンロード → Excel |

## 9. トラブルシューティング

### アプリが起動しない

| 症状 | 原因と対処 |
|---|---|
| `python` コマンドが見つからない | PATHが通っていない → Pythonを再インストール（PATHにチェック） |
| `streamlit` コマンドが見つからない | 仮想環境が有効化されていない → `venv\Scripts\activate` を実行 |
| ポートが使用中 | `netstat -an | find "8501"` で確認 → 別プロセスを終了するか別ポートを使用 |
| モジュールが見つからない | `pip install -r requirements.txt` を再実行 |

### 他のPCからアクセスできない

| 症状 | 原因と対処 |
|---|---|
| ページが表示されない | ファイアウォール設定を確認（セクション4-3） |
| 接続がタイムアウト | IPアドレスが正しいか確認（`ipconfig`） |
| 同じネットワークか確認 | クライアントとサーバーが同じサブネットにいるか確認 |
| `--server.address 0.0.0.0` を忘れていないか | 起動コマンドを確認 |

### データ関連

| 症状 | 原因と対処 |
|---|---|
| 報告が保存されない | `data` フォルダの書き込み権限を確認 |
| データが消えた | バックアップから `incidents.csv` を復元 |
| CSVが文字化け | Excel で開く際に UTF-8 を指定（「データ」→「テキストから」） |

## 付録: サーバーPCのIPアドレスを固定する方法

DHCPでIPアドレスが変わるとアクセスURLが変わってしまいます。
情報システム部門に以下を依頼してください：

1. **DHCP予約**（推奨）: サーバーPCのMACアドレスに固定IPを割り当て
2. **手動設定**: ネットワーク設定で固定IPを設定
   - 設定 → ネットワークとインターネット → イーサネット → IP設定の編集
   - 「手動」を選択 → IPv4をオン
   - IPアドレス、サブネットマスク、ゲートウェイ、DNSを入力

※ 設定値は情報システム部門に確認してください。
